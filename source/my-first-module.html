CP+B Front-End Developer Guide : Module

So you want to build a module?

Awesome! The module process helps you not only with creating a utility that the team will thank you endlessly for in future projects but also with building your own personal brand. It's your opportunity to biuld something cool and show it off/use it on future projects.

But it's more than that! This is also your own personal step into the open source world! Your opportunity to show the world what you're made of code wise.

What are the rules?

The only rules are that you follow the CP+B coding standards in order to contribute the module back to the CP+B Boilerplate. Other than that, no major 3rd party libs, work with that you have (don't make us use CoffeeScript, we don't wanna).

Where do I start?

Building a module is fairly easy. Just simple these simple steps:

  1. 1. Determine your Module's Purpose

    There a a plethora of modules which could be built. Think about what have you built several times and don't want to build again. Or, what has been built but you think you can build better. OR What has already been built well by someone else and you want to showcase and display the capabilities.

    Some possible suggestions to think about are:

    • Carousel

    • Form Validation

    • Form Styles

    • Or whatever else you'd like!

    To see if what you are building has been implemented, simply install the Yeoman CP+B Boilerplate Generator and run it to see if your module exists aready. Or ask a tech lead.

  2. 2. Install the Yeoman CP+B Module Generator

    Simply npm install the generator with this code in terminal

    npm install -g generator-cpb-module

  3. 3. Install the Yeoman CP+B Module Generator

    Simply npm install the generator with this code in terminal

    npm install -g generator-cpb-module

  4. 4. Create a new Module Folder

    In Terminal

    mkdir module-{{name of your module}} && cd module-{{name of your module}}

  5. 5. Let Yeoman Generate the Module-Boilerplate

    In Terminal

    yo cpb-module

    When prompted give the {{name of your module}}

  6. 6. Create/Update README

    Optimally we would like the direction of development to be based on the README. We would like to see a plan for a minimum viable product and what the next steps are in order to make this product better. This is a skill that some of you might already do but in case you don't it's useful to know. Please try to do this.

  7. 7. Phew! Finally you can start builing your module!

    Time to open the editor and dive into some code!

  8. 8. Done?! Already?!

    Congrats! Push that baby to github!

  9. 9. Time to add your module to the CP+B Boilerplate module options!

    To get started with that, clone the CP+B Yeoman Generator Repo and CD to the cloned repo, found here:

    https://github.com/cpbtechnology/US-Int-YO-CP-B

  10. 10. Now here's the tedious part:

    The CP+B Boilerplate is smart but not Omnipotant. We need to know how to use your module. We need to know what you added/changed to make your module work.

    To start out find the file within the CP+B Boilerplate Repo:

    app/templates/modules.json

    Open that file!

  11. 11. Okay that wasn't tedious but this is:

    Within the 'modules' array add a new Object. The object should look like this:

    {
    "name": "{{ name of your module }}",
    "value": "{{ name of user who owns the repo of the module }}@{{ name of the module repo }}",
    "cpb-module": [
    {
    "module": "{{ source of a file you want to move in your repo }}",
    "cpb": "{{ location/name of where we want the file you have selected above to live in the CP+B Repo }}"
    }, {
    "module": "{{ ANOTHER source of a file you want to move in your repo }}",
    "cpb": "{{ ANOTHER location/name of where we want the file you have selected above to live in the CP+B Repo }}"
    }
    ]
    }

    Now as you can see there is a lot of data to fill out, but the basic pieces of it is a name of the project, name of user that controls the module repo, the name of the repo the module exists and all of the files needed and paths to their destination. It's tedious but so friggin worth it!

  12. 12. Bask in the awesomeness that is, you.
